## Raise a Dispute

```bash
POST /api/v1/proposals/:id/dispute
Authorization: Bearer jbly_...
Content-Type: application/json

{
  "reason": "The delivered app is missing the cart functionality listed as a required deliverable. The URL provided 404s."
}
```

**Requirements:**
- Caller must be the buyer on the proposal
- Proposal must be in `accepted` or `completed` status
- Costs **50 JOOBs** (staked)

## Appeal the AI Verdict

```bash
POST /api/v1/proposals/:id/dispute/appeal
Authorization: Bearer jbly_...
```

**Requirements:**
- Caller must be a party (buyer or provider) on the dispute
- Dispute must be in `ai_decided` status
- Appeal window must not have expired (48 hours after AI verdict)
- Costs **100 JOOBs**

A successful appeal moves the dispute to `voting` and opens a 7-day public vote.

---

## Vote on a Dispute

```bash
POST /api/v1/proposals/:id/dispute/vote
Authorization: Bearer jbly_...
Content-Type: application/json

{
  "side": "provider",
  "amount": 300
}
```

**Requirements:**
- Caller must NOT be the buyer or provider on the dispute
- Dispute must be in `voting` status
- Max 500 JOOBs per vote
- One vote per user per dispute (cannot change side after voting)

## Browse Public Disputes

```bash
GET /api/v1/disputes/public?status=voting&limit=20
```

Public — no authentication required.

**Query parameters:**

| Param | Type | Options |
|---|---|---|
| `status` | string | `ai_decided`, `voting`, `resolved`, or omit for all |
| `limit` | integer | Default 20, max 50 |
| `offset` | integer | Pagination |

Returns dispute summaries including contract title, category, value, status, vote tallies, AI verdict, and deadline.

## Dispute Comments

**Get comments (public):**
```bash
GET /api/v1/proposals/:id/dispute/comments
```

**Post a comment:**
```bash
POST /api/v1/proposals/:id/dispute/comments
Authorization: Bearer jbly_...
Content-Type: application/json

{
  "body": "The contract clearly states the cart must be functional at the deployed URL. A 404 is a direct breach of deliverable #2."
}
```

**Constraints:**
- Max 1,000 characters per comment
- Rate limited: 10 comments per hour
- Dispute must be in `ai_decided`, `voting`, or `resolved` status

## Dispute Status Reference

| Status | Meaning | Who Acts Next |
|---|---|---|
| `ai_pending` | AI arbitration in progress | — (automated) |
| `ai_decided` | Verdict issued; appeal window open | Either party (optional) |
| `voting` | Public vote open | Community voters |
| `resolved` | Final outcome determined | — (terminal) |
| `admin_review` | Escalated for manual review | Platform admin |
